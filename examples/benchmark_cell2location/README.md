# Cell2location Spatial Deconvolution Benchmark Example

This example demonstrates how to benchmark an `analysis` skill that performs
spatial transcriptomics deconvolution with
[cell2location](https://github.com/BayraktarLab/cell2location).

It follows the **Darwin** testing strategy:

| Stage | Level | Data | Purpose |
|-------|-------|------|---------|
| 0 | Unit | Tiny synthetic (10 spots, 3 cell types) | Smoke test: output shape and basic validity |
| 1 | Component | Small synthetic (50 spots, 5 cell types) | Mock regression: RMSE against golden proportions |
| 2 | Integration | Medium synthetic (200 spots, 8 cell types) | Correlation and stability checks |
| 3 | Release | User-supplied real Visium | Completion-only acceptance, requires GPU |

## Structure

```text
examples/benchmark_cell2location/
├── SKILL.md                                            # Example skill documentation
├── benchmarks/bio-spatial-deconvolution-cell2location/
│   ├── registry.yaml                                   # Per-skill benchmark registry and suites
│   ├── tasks/deconvolution.yaml                        # Task specification
│   ├── data/
│   │   ├── tiny/                                       # Level 0 dataset
│   │   ├── small/                                      # Level 1 dataset
│   │   ├── medium/                                     # Level 2 dataset
│   │   └── real_visium/                                # Level 3 dataset (user-supplied)
│   └── expected/                                       # Golden proportion matrices
├── runner.py                                           # Custom task runner for deconvolution
├── scripts/
│   ├── generate_data.py                                # Generate synthetic Visium + reference data
│   └── run_c2l.py                                      # Example skill code (Agent-generated)
└── README.md                                           # This file
```

## Quick Start

1. Generate synthetic data:

```bash
python examples/benchmark_cell2location/scripts/generate_data.py
```

2. Run the full Darwin pipeline up to integration level:

```bash
test-skill --mode gradual \
    --skill bio-spatial-deconvolution-cell2location \
    --registry examples/benchmark_cell2location/benchmarks/bio-spatial-deconvolution-cell2location/registry.yaml \
    --max-level 2 \
    --output-dir examples/benchmark_cell2location/ci-output
```

3. Run a single level manually:

```bash
test-skill --mode single \
    --skill bio-spatial-deconvolution-cell2location \
    --registry examples/benchmark_cell2location/benchmarks/bio-spatial-deconvolution-cell2location/registry.yaml \
    --level 1 \
    --output examples/benchmark_cell2location/results_level1.yaml
```

4. Use the example skill code instead of the fallback runner:

```bash
test-skill --mode single \
    --skill bio-spatial-deconvolution-cell2location \
    --registry examples/benchmark_cell2location/benchmarks/bio-spatial-deconvolution-cell2location/registry.yaml \
    --level 1 \
    --code examples/benchmark_cell2location/scripts/run_c2l.py \
    --output examples/benchmark_cell2location/results_level1.yaml
```

## Notes

- The fallback runner uses a simple NNLS deconvolution. A real cell2location
  implementation would be generated by an Agent from `SKILL.md` and executed
  via `--code`.
- Level 3 (`real_data: true`) is marked `requires_gpu: true` and is checked for
  completion only, not scored, because real Visium data is expensive to run.
- The `darwin` suite is used by `test-skill --mode gradual` to progress from cheap unit tests
  to expensive release acceptance.
